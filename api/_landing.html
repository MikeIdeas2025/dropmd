Why Markdown

Your LLM already speaks it

Models are trained on enormous amounts of Markdown. Give them a format they know and the answers get better.

Better answers

A heading tells the model what a section is about. A wall of text extracted from a PDF does not.

Structure survives

Copy-pasting from Word flattens everything. Markdown keeps the hierarchy that gives your prompt context.

Cheaper prompts

Markdown adds almost no markup, so the same document costs fewer tokens than HTML or raw extracted text.

API

There's an API too

The same endpoint that powers this page. Send raw bytes, get JSON back. No key needed while in beta.

# Convert any file to Markdown
curl -X POST --data-binary @report.pdf \
     -H "x-filename: report.pdf" \
     https://dropmd.vercel.app/api/convert

# → { "ok": true, "markdown": "# …", "tokens_estimate": 1234 }
FAQ

Questions people ask

Is it really free?

Yes. No account and no limit on conversions while DropMD is in beta. A paid tier for bigger files and batch conversion may come later, but the free converter stays.

What happens to my files?

They are converted in memory on a serverless function and discarded right after the response. Nothing is written to disk, stored or logged.

Why the 4MB limit?

It's the request size limit of the serverless infrastructure this runs on. If you need bigger files, open an issue on GitHub: it's the first thing a paid tier would unlock.

What's the engine behind it?

Microsoft's open source MarkItDown, the same library used in LLM data pipelines. DropMD is a web UI and API on top of it, so you can use it without installing Python.

Is the output good for human readers?

It's made for machines. The Markdown is clean and structured, but the goal is LLM consumption, not a print-ready document.

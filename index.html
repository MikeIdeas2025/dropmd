Why Markdown for LLMs?

✦

Models speak it natively

Mainstream LLMs are trained on huge amounts of Markdown — they parse headings and tables in it far better than raw PDF text dumps.

✦

Structure survives

Copy-pasting from Word or PDF flattens everything into a wall of text. Markdown keeps the hierarchy that gives your prompt context.

✦

Fewer tokens, lower cost

Markdown is close to plain text: minimal markup means fewer tokens per document — cheaper and faster prompts.

FAQ

Is it really free?

Yes. DropMD is free while in beta — no account, no limits on the number of conversions. A Pro tier (larger files, batch conversion, API keys) may come later; the free converter stays.

What happens to my files?

They are converted in memory on a serverless function and discarded immediately. Nothing is written to disk, stored or logged.

Why is there a 4MB limit?

It's the request-size limit of our free serverless infrastructure. Need bigger files? Open an issue on GitHub — it's the first thing the Pro tier will unlock.

What's the engine behind it?

Microsoft's open-source MarkItDown — the same library used in LLM data pipelines. DropMD wraps it in a zero-setup web UI and API.

Is the output perfect for human readers?

It's optimized for machines, not for print. The Markdown is clean and structured, but the goal is LLM consumption — not pixel-perfect document fidelity.
